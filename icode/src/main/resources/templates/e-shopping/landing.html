Welcome to E-shopping

Display Products

Welcome to building the e-shopping application. In order to complete this application, you will define new objects, create implementation logic, and develop new methods. The pseudocode is already provided for most of the application. You will need to make use of conditional statements, loops, as well as the session.

Phase 1:

During this phase you will create the required objects for this application to function.

Product

Create a Product POJO -- ensure it is stored within an appropriate package

    What a Product should contain

  • id of type int
  • name of type String
  • price of type BigDecimal
  • getter and setters
  • toString, hashCode, and equals

ShoppingCart

Create a ShoppingCart POJO

    What a ShoppingCart should contain

  • products as a List
  • getter
  • methods to add and remove products

ProductService

Create a ProductService class, this class will be used to communicate between the Controller and Repository. It will handle any additional processing when necessary.

    What a ShoppingCart should contain

  • repository of type ProductRepository -- it will be @Autowired through the constructor
  • getAllProducts() - this method takes in no arguments and returns a list of products -- calls the appropriate method in the repository
  • getProductById() - this method takes in a product id, passes it to the appropriate repository method and returns the found Product object

Phase 2:

Build out ProductRepository

    This class has the following attributes and methods

  • products - a list of products
  • sampleProducts() - method that create a collection of products and is called in the constructor to set a collection of products to the attribute in the class
  • getProducts() - returns list of products
  • getProductById() - takes in an id and returns Product with matching id
  • addProduct() - takes in a product and adds it to the list of products -- does not return anything
  • updateProduct() - takes in a product that exists in the list and its updated information. The updated product replaces the original
  • deleteProductById() - takes in an id and deletes the associated product from the list

Ensure EShoppingController has dependencies

    This class has the following attributes

  • productService - ProductService
  • context - ApplicationContext

    • Both dependencies should be @Autowired through the constructor

Phase 3:

Add implementation logic for EShoppingController methods

  • goDisplayProducts()
  • addToCart()
  • removeFromCart()
