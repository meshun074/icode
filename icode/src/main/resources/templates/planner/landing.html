Planner Application

Welcome to the Planner application. This application will be used to track and update a list of Todo items. Each Todo item will contain a collection of tasks, will have a topic, and a due date. This application will allow users to create new todo items, view the items they have stored, update their Todo items, delete their items while also adding search and sorting functionalities. This document will provide you with the business requirements of the application and will also provide hints and code if needed.

Allow users to delete and search through Todo Items Hints

In order to achieve this task, you will need to complete the following steps.

  • Allow your users to delete Todo items from the details page.
  • Allow your users to search Todo items displaying items:
    • View items by topic
    • View items between date range
    • View items ordered by due date
    • View item(s) that contain specific tasks
  • Application flow of information
      • Delete Todo item
    • Todo details contains a link that passes todo id to Controller
    • Users are redirected back to details to confirm they want to delete the item
    • If yes, the id is passes back to the Controller
      • Controller passes id to Service which passes it to the Repository
      • Repository removes Todo item from collection
      • Controller returns user to the home page
    • If no, the user is returned back to todo-details, no change.
      • Search Todo items
    • Allow users to navigate to the search page from the home page
    • Create a form that allows users to view a list of items based on topic
      • Form passes topic to Controller
      • Controller passes topic to Service which passes it to the Repository
      • Repository returns all items of the specified topic
      • Controller returns user to the search page displaying list
      • If no items are present, 'No items found' message is displayed
    • Create a form that allows users to input a date range
      • Form passes range to Controller
      • Controller passes range to Service which passes it to the Repository
      • Repository returns all items between the specified range
      • Controller returns user to the search page displaying list
      • If no items are present, 'No items found' message is displayed
    • Create a link that allows users show priority items
      • Controller calls Service to retrieve information from the Repository
      • Repository returns all items sorted by due date
      • Controller returns user to the search page displaying ordered list
    • Create a form that allows users to search items by tasks
      • Form passes task to Controller
      • Controller passes task to Service which passes it to the Repository
      • Repository all items that contain the specified task
        • E.g., if a task was "cook" any item that contains a task with "cook" inside of it will be included in the list
        • "cook" -> "Cooking pies", "cook meal prep", "Thanksgiving cooking"
      • Controller returns user to the search page displaying list if a list greater than one is returned
      • Controller returns user to the search page displaying singular item details if a list of size 1 is returned
      • If no items are present, 'No items found' message is displayed
