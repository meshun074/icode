What does object-oriented mean?

OOP is a method of developing programs where code is encapsulated in different files or data types. This allows code to be focused and follow SOLID standards of coding.

  S - Single Responsibility Principle - responsible for one thing
  O - Open Closed Principle - open to extension, closed for modification
  L - Liskov Substitution Principle - Superclasses should be replaceable by subclasses
  I - Interface Segregation Principle - interfaces must remain focused
  D - Dependency Inversion Principle - depend on abstractions instead of concretions

Several different types of Objects (data types) exist within Java, the most commonly used include: Class, Interface, and ENUM.
Classes defines the blue print of an object. Classes that only store persistable information can be called POJOs (Plain Old Java Objects).
  • Attributes (fields) - what a class can store
  • Methods (behaviors) - functions a class can perform
  • Access modifiers - default private protected public
Interfaces define behaviors.
  • Attributes (if present) - public static final
  • Access modifiers - public - do not need to define
ENUMs define attributes.
  • Access modifiers - public - do not need to define
  • Static and Final
Object-oriented programming stands on four pillars.

  1. Encapsulation
    The act of storing data (e.g., attributes, fields, variables) and methods (behaviors) in one unit.
    public class Person {
      private String name;
      private int dateOfBirth;

      // Getter example
      public String getName(){
        return this.name;
     }
    }
  2. Inheritance
    The act of gaining attributes and/or behaviors from other objects.
    • Data types can only extend from one superclass
    • Classes can implement n number of interfaces
    public abstract class Student {
      private String[] classes;
      public abstract void takeExam();
     }
    }

    public interface IPayTaxes {
      void payTaxes();
    }

    public class Person extends Student implements IPayTaxes {
    }
  3. Abstraction
    The act of hiding the implementation details from user; only providing minimum necessary code.
    public class Runner {
      public static void main(String[] args){
     Person person= new Person();
      person.payTaxes();
      }
    }
  4. Polymorphism
    The ability for methods to take many forms.
    • Overloading - same method name, different parameters (order, number, type) - compile time
    • Overriding - inherited method, different implementation - runtime

What is Thymleaf?

Server-side template engine used to process HTML, XML, JavaScript, CSS and even plain text. The goal of Thymleaf is to create maintainable HTML templates

To add Thymleaf to HTML, you add 'xmlns:th="http://thymeleaf.org' to the openning html tag.

View Available Thymleaf expressions

  • th:text- will insert text associated with given attribute (line 241)
  • th:href- associates a link with the desired GetMapping location (line 198)
  • th:action- associates a form with the desired mapping location (line 236)
  • th:each- iterates through a collection of objects e.g., 'th:each ='pet : ${pets}'' (trail-home.html line 36)
  • th:object- binds an object to a specific section or form (trail-create.html line 26)
  • th:field- accesses a field of an object (trail-create.html line 28)
  • th:if- conditionally displays information (${ifVariable} eq true, false, null, etc.) (line 199)
  • th:fragment- creates a reuseable block of code (nav.html line 9)
  • th:insert- inserts a reuseable block of code (line 26)

You can display information from the back-end using the Model or Session

  • Information stored on the model will be available for one transaction -- access by ${attributeName}
  • Information stored on the session will be available so long as the session is valid -- access by ${session.attributeName}
